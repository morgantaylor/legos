Accordion

There are two types of accordions, an accordion, and accordion--panel. They both can have very different use cases. The floating navigation items on the left side of the screen are accordions.

The component is simple with a few enhancements depending on the intended use. By default, the accordion is in a closed state and spans the entire width of its parent container. Add a max-width property to the wrapper element to control its width.

Attributes:
  • aria-label : the toggle input will never have a corresponding label, be sure to have an aria-label.

*Note: The i element is semantic and is used to animate the arrows' opened and closed state. Without it, the user wouldn't know the component has a toggle for states.

Basic

The default accordion component looks like it's floating. The background's transparent on both title and container.

  </div>
</div>

Panel

To make an accordion a contained object, giving it a border and background color, add --panel to the accordion class. That's it! Now the content inside the accordions' container looks like it's contained inside.

Adding the --panel class to an accordion

  • Will round the border-radius
  • Gives the accordion a background-color
  • Adds consistent padding to the accordion contents

Skins

All basic skins are available for this component.

Add the skin to the accordion component, inside the .accordion__wrapper or it will not work.

*Note: This is only intended for the --panel accordion. The default accordions' purpose is to look like it's floating.
