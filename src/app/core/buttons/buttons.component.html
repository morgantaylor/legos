Button

When creating a button use a <button> or <a> element for accessibility purposes. There are three different types of buttons.

Attributes:
  • type : "submit", "reset", "button"
  • aria-label : defines a label for the element (only used when label text not visible on screen)
  • name
  • value

States

Buttons have three states besides the standard button state.

They are :hover, :active, .active and :disabled.

Skin

Buttons are one of the components that have their own dedicated skin. It is very similar to global skins but with additional characteristics.

  • btn--{{ color }}
  • btn--outline-{{ color }}

Filled

Filled buttons have colored borders, backgrounds, and white font color.

Demo:
Markup:
<button type="button" class="btn btn--blue">...</button>
<button type="button" class="btn btn--green">...</button>

Outlined

Outlined buttons are the reverse of filled buttons. Their font and border have color while the background is white. Otherwise, they are the same as filled buttons.

Demo:
Markup:
<button type="button" class="btn btn--outline-red">...</button>
<button type="button" class="btn btn--outline-orange">...</button>

Links

This element is not a button element, it is infact an a element.

While not specifically a button (it does not get the .btn base), links are still in the button category because they use the same skin and hover actions. A link is an inline element with text that takes up only the amount of space it needs, unlike buttons who have padding.

Outline skins are not available for links.

*Note: Anchor elements do not have the :disabled state. Use the aria-disabled="true" attribute to style a disabled anchor element the same as a disabled button.

Demo:
link {{ color }} disabled link {{ color }}
Markup:
<a class="link--blue">...</a>
<a class="link--green" aria-disabled="true">...</a>

Block Buttons

Block buttons take up the entire width of their parent container.

Demo:
Markup:
<button type="button" class="btn btn--block">Default Size</button>
