Accessibility

In compliance with Web Content Accessibility Guidelines (WCAG), 2.0these components aim to be accessible to assistive technologies and promote best practices for developers.

ARIA: Roles, states, and properties

Being ARIA compliant has become standard. This includes having the correct value for the role of an element, and aria-label attributes alongside the basic element attributes, id, for, type, title, name, and value.

Images

Always provide alternative text for images. The alt attribute is not only used for assistive technologies like screen readers. It is also useful if a user has a slow internet connection and cannot view the image or if there is an error in finding the image.

<img src="cat.png" alt="A picture of a orange cat sleeping on a brown chair">
Colors

Colors are critical to a website. There are several different impairments that a user can have. As designers and developers, we need to accommodate the blind, color blind, and those with partial impairments. It is also best practice to ensure the contrast between text and background colors is enough for those with color blindness. There are several free tools available that will scan your page and compare text and background.

  • Colorblind Web Page Filter - examples of different levels of colorblindness
  • Color Contrast Accessibility Validator - scans websites to verify the contrast and ensure better accessibility
Links

Like images, it is important to add alt attributes on anchor elements, describing what it is. If linking to a document, for example, a PDF, tell the user that in the description. Users want to know what they will be opening ahead of time. Always confirm your links work before your website goes live and go where they are supposed to. You wouldn't want a user to click a link that goes nowhere.

<a href="http://wwww.gosomewhere.com" alt="A link to a new page" rel="noreferrer">
